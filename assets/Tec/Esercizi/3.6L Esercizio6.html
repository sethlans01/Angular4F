Esercizio 6 - Videogame Leaderboard

Realizzare la seguente webapp in Angular

La webapp ha lo scopo di rappresentare una lista di giocatori di un videogioco e la loro posizione nella classifica competitiva.

  1. La webapp deve essere realizzata su una singola pagina.
  2. Il videogioco dovete sceglierlo voi, a patto che sia un videogioco multiplayer.
  3. I videogiocatori devono far parte di una squadra competitiva famosa.
  4. Nella pagina web devono essere presenti un titolo, un’immagine o un logo del videogioco, una breve descrizione dell’obiettivo del gioco e, infine, una lista di 10 giocatori almeno.
  5. La lista di giocatori può essere disposta in qualsiasi modo, quindi sia una lista verticale, che orizzontale che una griglia vanno bene.
  6. Per ogni giocatore dovranno essere mostrate all’interno di una card le seguenti informazioni:
  1. Nome utente
  2. Età
  3. Nazionalità
  4. Ruolo / personaggio preferito nel videogame
  1. Inoltre nella card dovrà essere presente un bottone che rimanda ad una pagina web (che non dovete realizzare voi, ma dovete trovare su internet) che contiene una notizia che riguarda il giocatore.
  2. I dati dei vari giocatori e il link alla pagina esterna dovranno essere memorizzati all’interno di un array di oggetti nel file Typescript.
  3. La redirezione alla pagina di ogni giocatori mediante il bottone dovrà essere realizzato con l’event binding di Angular ( usare (click)=”........” )
  4. Infine, dentro ogni card dovrà essere presente un badge (anche chiamato label o tag) che deve contenere una bandierina della nazione del giocatore.
  1. Il punto 10 dovrà essere realizzato con la funzione NGSWITCH di Angular
  2. Per comodità conviene usare delle emoji per le bandierine (https://getemoji.com/)
  3. Cosa è un bagde / label / tag:
    https://www.w3schools.com/bootstrap/bootstrap_badges_labels.asp
    https://semantic-ui.com/elements/label.html
  1. In base al ruolo / personaggio preferito lo sfondo della card deve essere di colori diversi
  1. Questa funzionalità dovrà essere realizzata con la funzione NGCLASS di Angular
  2. Per semplificare il lavoro, scegliete giocatori con ruoli / personaggi preferiti di modo tale da non avere troppe categorie
  1. Il progetto deve essere committato e pushato su una repo di GitHub
